~/.gitconfig の最低限の設定

どうせローカルでしかgitを使わないよ、と思っていても、そのうちGitHubなんかを使いたくなるかもしれないし、一応、ユーザ名とメールアドレスは、公開してもさしつかえのないものを設定しておくのが安心。

$ git config --global user.name 公開してもさしつかえのない名前 $ git config --global user.email 公開してもさしつかえのないアドレス

いちいちコマンドを打つのが面倒だったら、~/.gitconfigを直接編集してもよい。下記は、カラー設定と、エディタ設定と、常に管理対象から外しておきたいものを指定するための設定と、いくつかのエイリアスの設定の例。以下のように設定したうえで、~/.gitignore_globalに.DS_Storeと書いておけば、「知らぬ間になぜか.DS_Storeができていて、それがgitの管理対象に入っている」という事態にならない筈。なお、エディタは、最初は大好きなmiにしたかったんだが、パッケージ型のエディタであるせいか、うまく呼び出せなかったので、emacsを設定してみた。

[color] ui = auto [core] editor = emacs excludesfile = ホームディレクトリへのパス/.gitignore_global [alias] com = commit br = branch st = status co = checkout sh = show

git diffのカラー表示のための設定については、別ページに書いた。

追記: エディタの設定 (mi も指定できるみたい)

テキストエディットや Sublime Text をエディタとして指定する方法について書いてあるページを見つけたので、それを参考にして、少し試行錯誤してみた。 以下のようにすれば、エディタとして mi を指定することもできるようである。

[core] editor = open -a mi.app -n -W

-a オプションでアプリケーションを指定し、-n オプションで、mi が起動済みの場合には別インスタンスで mi を起動するようにしておく。そして、-W オプションで、その新たに起動した mi が終了するのを待つようにする (それによって、たとえば、mi が終了するまでは、コミット・メッセージの入力待ち状態になる)。 なお、/Applications/mi.app/Contents/MacOS/mi のような実行パスを指定するのが気持ち悪かったので、-a オプションを使う形にしてみた。

ただ、常に mi の起動を待つ時間 (しっかり体感できる程度には長い) が発生してしまうので、使い勝手としては、ちょっと微妙かもしれない。
