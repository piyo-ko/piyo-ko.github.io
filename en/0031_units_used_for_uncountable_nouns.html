英語あれこれ: 数えるときに使う単位

英語あれこれ: #不可算名詞 #単位

不可算名詞を数えるときに単位として使える単語も色々あるけれど、情報通信系でよく見かけるものを思いつくままに挙げておく。なお、可算名詞を数えるときに、この種の単位を使うこともある。

articlean article of 〜 は、汎用性の高い数え方だと思う。たとえば、equipment を数えるのにも使える。
block辞書を見ると、「符号」の意味で使っている code は (文脈によっては) 可算のようだが、machine code や source code などは不可算名詞。まあ、言われてみれば、ソースコードなんていくらでも恣意的な切り取り方ができるわけだから、「ここからここまでが一個だよね」という誰もが同意する明示的な範囲は決められないということで、不可算名詞なのも納得である。ということで、数えたいときは、a block of code などと数えると便利。もちろん、a piece of code でもよい。
chunka chunk of code という数え方も可能なようである。でも、ある程度大きなまとまり、というイメージかな?
copy複製したものに関して (あるいは原本と複製を区別せずに同一内容のものに関して)、「一部」とか「一件」と数えるのに、a copy of 〜 という言い方を使う。下記の instance のところに書いたのと同様な文脈でも重宝すると思う。
cyclethe number of cycles of 〜 per second みたいな形で、何か繰り返し生じる物事の回数を数えるときに使う。なお、a cycle of 〜 は、「〜の循環」「〜の連鎖」「〜の繰り返し」「一連の〜」などと訳せる (これは、文脈によって、可算名詞をとることも不可算名詞をとることもあるみたい)。
instanceコピーとかクローンとか複製とかインスタンス化とか、呼び名はどうあれ、とにかく「同じものがいくつもあるよ」という状況のときに、それらのものについて、「元は同じ一つのもの」「でも今や複数の別々の実体が存在している」という二面性を明確に表すために便利なのが、multiple instances of the same 〜 のような言い方だと思う。
iteman item of 〜 も、汎用性の高い数え方。a data item のように、(今やほとんど不可算名詞の単数形のような扱いをされる) data の後ろにくっつけて、可算の名詞句にするような使い方もできる。
piecea piece of 〜 は、極めて汎用性が高い。information, data, code, software, hardware, firmware, machinery, circuitry, text などなど、多くのものを piece で数えることができる。
seta set of 〜 は、「〜の集合」という数学的な意味を表す場合は、可算名詞の複数形とともに使う。「一揃いの〜」「〜一式」といった意味を表すときは、可算名詞でも不可算名詞でも使えるような気がする。a set of code とか。なお、data を可算名詞句化するのには、上記の a data item という言い方のほかに、a dataset (または a data set) という言い方があるが、dataset はデータのまとまり全体のことで、data item はその中の一項目、という違いがある。
sheeta sheet of paper のように、薄っぺらいものを数えるときに使う。半導体製造についての説明なんかだと、層を重ねていく感じだから、sheet という語もわりと出てくるように思う。
time何かが起きる回数のことは、the number of times that 〜 と表現できる。繰り返し回数については、the number of iterations とか the number of repetitions といった表現も可能。
turnコイルの一巻き、ロープの一巻き、ねじの一回転などを指すときに a turn of 〜 という言い方をする。コイルの巻き数は the number of turns of the coil などと表せる。
