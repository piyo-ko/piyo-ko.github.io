英語あれこれ: 「閾値」に関する語彙

英語あれこれ: #閾値 #threshold #subthreshold #thresholding #thresholded #unthresholded

名詞としての threshold と「閾値」

「閾値」の訳語は、threshold 一択と考えてよい。 ときどき「値」につられて threshold value と訳してあるのを見るが、これは冗長だろう。

subthreshold という派生語

threshold の派生語の subthreshold は、「サブスレッショルド」「閾下」などと訳すようだが、「サブスレッショルド領域」「サブスレッショルド特性」「サブスレッショルド・リーク電流」などの特定の用語において形容詞的に使う場合がほとんどではなかろうか。

threshold の動詞的な派生用法

threshold を動詞とみなしているような使い方もあるが、実際上は、動名詞か過去分詞形での用法がほとんどではなかろうか (ごくたまに、「閾値に至る」「閾値を跨ぐ」「閾値を超える」のような自動詞として使う事例があるようだが)。

thresholding という動名詞は、グレースケール画像から白黒二値画像を生成することを指すのに使われる。 広く一般に閾値によって二分する処理についても thresholding と言えるかどうかはよく分からないが、言えそうな気もする。どうだろう。

thresholded という過去分詞形は、形容詞的に使われる。 これの対義語は unthresholded である。 ただ、これらを和訳する際には、少し注意が必要なように思う。 何らかの変数 x (たとえば電圧など) と、それに関する閾値 T があるとして、訳語選択の際には以下のようなことに注意を払うのが良いかと思う。

  • x ≤ T の場合と T < x の場合に分けるのか。それとも、x < T の場合と T ≤ x の場合に分けるのか。つまり、閾値をどちらの範囲に含めるのか。それによって、「〜以上」「〜以下」「〜を超える」「〜未満」を適宜使い分けねばならない。 現実問題としては、ぴったり x = T の場合をどちらの範囲に含めようと何の違いもないかもしれないが、翻訳の場合は、きちんと訳語を使い分けたい。
  • x の変化という動的な現象について記述しているのか。それとも、閾値 T で分けられた二つの領域のどちらに x が含まれるか、という静的な状態について記述しているのか。 たとえば、静的な文脈の場合は「閾値到達後の」ような訳し方は好ましくないかもしれない (「到達」という「動作」を含意してしまうため)。
  • 変化について記述している場合、x の増加と減少のいずれについて記述しているのか。 減少について述べる場合には、「閾値 T を越えた (超えた)」と書くと、誤解を生じかねない。 なぜなら、「T < x の状態から、x が段々減って、ついに x が T を乗り越えて、x < T となった」ことを表したいのか、それとも「T < x」という最初の状態を表したいのか、曖昧なためである。「越えた」でなくて「過ぎた」としても、誤解を招くかもしれない。 特に、x が負のときは、x が小さくなるほど |x| は大きくなるので (別ページも参照)、全般的に原文・訳文ともに混乱しがちな気がする。
  • 変化について記述する文脈において、その変化は、一度閾値を跨いだら何か不可逆なことが生じるような種類のものか。それとも、可逆変化について述べているのか。この点が直接的に訳語に影響することはなさそうにも思うが、一応この点にも気をつけて推敲したい。 たとえば、可逆変化の場合に、x が一旦閾値 T を跨ぐ変化をした後、今度は逆向きに変化して再度閾値 T を跨いで元に戻ってくるかもしれず、その戻った後の状態を指して unthresholded を使うかもしれない。この場合の unthresholded については、「閾値に至る前の」などと「前」を使って訳すと、「時間的に後なのに『前』とはこれ如何に」という余計な疑念を招きかねない。
  • 閾値 T で分けられた二つの領域のうちの、どちら側が所望の (目的の) 状態なのか。あるいは、どちらが目的、といった観点は特にないのか。訳語には影響しないかもしれないが、文意を正確につかむためにも、一応気をつけておきたい。

以上のようなことに注意を払っていると、がんじがらめになって「どの訳語も今ひとつ」と思えてきそうだが、一応、考えうる訳を挙げておく。文脈に応じて適切なものを選ぶとよいと思う。

thresholded は、「閾値以上の」「閾値を超えた」「閾値を跨いだ」「閾値到達後の」「閾値に至った」「閾値を過ぎた」「閾値を乗り越えた」「閾値の向こう側の」などと訳せる。 また、unthresholded は「閾値以下の」「閾値未満の」「閾値を超えない」「閾値に到達する前の」「閾値に至っていない」「閾値の手前側の」などと訳せる。

文脈から判断して、誤解や問題が生じないようなら、単純に thresholded を「閾値以上の」と訳し、unthresholded を「閾値未満の」と訳すのが分かりやすいだろう。 しかし、(ほとんどありえないとは思うが) もしかしたら「初期状態では T ≤ x であり、unthresholded はその初期状態での T ≤ x という条件を指しており、x を T 未満にまで減少させることで所望の状態に至るので、thresholded とは x < T の領域を指す形容である」という文脈もあるかもしれない。 よって、文脈を考えずに一律に「閾値以上の」「閾値未満の」と訳すのは危なっかしい。
