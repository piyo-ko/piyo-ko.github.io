英語あれこれ: 対になっている動詞

英語あれこれ: #動詞 #対義語

対になっている動詞なんて山ほどあるわけだが、情報通信系でよく使うものを、とりあえず思いつく範囲で列挙してみる。

en- / de- 型

encapsulatedecapsulate「カプセル化する」の方はよく使う (訳し方については別のページにも書いた)。decapsulate という単語は存在するのだが、encapsulate と対にしてよく使うかというと、あまり使用例を見ないような気がする。というか、辞書を見る限り、decapsulate は外科用語っぽい。
encodedecode「符号化する」と「復号する」。
encryptdecrypt「暗号化する」と「復号する」。なお、ほとんどの場合、encipher と decipher は使うべきではない。
enqueuedequeue「キュー (の末尾) に追加する」と「キュー (の先頭) から取り出す」。キューではなくてスタックが対象だと、動詞は push と pop を使う。リストや文字列の操作だと、prepend A to B (Bの先頭にAを追加する) と append A to B (Bの末尾にAを追加する) と concatenate A and B (AとBを連結する) を使う。と、このように、特定のデータ構造と動詞が分かち難く結びついていることがある。push だって、スタックとは別の文脈なら、その対義語が pull だったりするし (情報配信などで「プッシュ型」と「プル型」と言ったりするし、git のコマンドでも push と pull は対義語的だし)。

- / de- 型

compressdecompress「圧縮する」と「解凍する」。「解凍」ではなく「伸張」または「展開」を使うこともある。
coupledecouple「結合する、連結する、接続する、つなげる、関連づける」と「切り離す、分離させる」。
identifyde-identify「識別する、特定する、同定する、突き止める、見分ける」(色々な文脈で使える言い方) と「無名化する」(プライバシの文脈で使う言い方)。後者は、de-identified data などと受け身で使うことが多いのかな? ただ、「無名化する」=「連結不可能匿名化する」=「元は誰のデータだったのかを、たとえ他のデータと突き合わせても突き止めることができないように、何らかの処理を施す」という理解で良いのかどうか、大変不安である。pseudonymize (仮名化する) とか anonymize (匿名化する) とか、似た概念があり、ややこしくて把握しきれていない。 なお、場所に関することなら、identify の代わりに locate を使えることもある (位置を突き止める、といった意味で locate を使える)。
interleavedeinterleave「インタリーブする」と「デインタリーブする」。
modulatedemodulate「変調する」と「復調する」。
selectdeselect色々と訳しようはあろうが、便利なのは「選択する」と「選択解除する」。文脈によっては、enable と disable に置き換えられるかも。deselect の他に、unselect も使える。ただ、受身の形にして名詞句を修飾する場合には、unselected だと「まだ選択されていない、未選択の」という雰囲気で、deselected だと「選択が解除された」という雰囲気のような気がする。どうだろう……。
serializedeserialize「シリアライズする」と「デシリアライズする」。これについては、別のページにも少し書いた。
spreaddespread(スペクトラム拡散の文脈で)「拡散する」と「逆拡散する」。

- / dis- 型

chargedischarge「充電する」と「放電する」。双方とも、これ以外の意味も色々とある。
connectdisconnect「接続する」と「切断する、遮断する、接続を切る」。
continuediscontinue「続く、続ける」と「途絶える、中止する」。discontinue という動詞よりも、discontinuity (不連続) という名詞や discontinuous (不連続な) という形容詞の方が (数学の分野などで) よく見るかもしれない。

- / in- 型

activateinactivate「起動させる、作動させる、活性化する」と「作動しないようにする、不活性化する」。deactivate という言い方もある。
validateinvalidate「有効にする、妥当性を立証する」と「無効にする、間違いだと立証する」。

- / un- 型

comment outuncomment「コメントアウトする」と「アンコメントする」。
foldunfoldGUI の文脈で、メニューやリストに関して「折りたたむ、しまう」と「展開する」という意味で使うように思うんだが、collapse と expand の方がよく使うかもしれない。どうだろう? collapsible tree とか呼ぶみたいだしな……。
installuninstall「インストールする」と「アンインストールする」。
selectunselect上記の select / deselect の項を参照。

a- / di- または de- 型

associatedissociate「関連づける」と「関連づけを解除する」。
attachdetach「取り付ける」と「取り外す」。

その他のパタン

enabledisableコンピュータの機能などを「有効にする、作動させる」と「無効にする、作動しないようにする」。
setreset文脈によって色々に訳せるけれど、「セットする」と「リセットする」とするのが楽だし問題も生じない、という場合が多いかな。これについては、別のページにも書いた。
