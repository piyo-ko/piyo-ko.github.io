英語あれこれ: #数値 #大小関係

何かを表す二つの数値 p と q があるとして、 「p < q であること」と 「|p| < |q| であること」と 「p より q が良いこと」は、違うことである。これら三つの事柄を混同してはいけない。 たとえば、負の値は、値の大小と、値の大きさ (絶対値) の大小が逆なので、lower than, higher than, greater than などの言葉に気をつけないといけない。でも、紛らわしい書き方とか、混乱した書き方が、ときどき見受けられる。

もちろん、場合によっては、 「p も q も正である」という前提があるために「p < q と |p| < |q| は同値」と言えたり、逆に 「p も q も負である」という前提があるために「p < q と |p| > |q| は同値」と言えたりする。でも、「前提を明示しなくてよい」ということにはならないし、「|p| < |q| と言うべきところで p < q と述べて、それによって混乱を招いても構わない」ということにはならない。

適切な動詞を選ぶには、シソーラスを見るとよいと思うが、とりあえず、数値と大小関係にまつわる表現と、注意事項とを、思いつく範囲で挙げておく。

  • p < q は、p is less than q とか p is smaller than q とか q is greater than p とか q is larger than p のように表現できる。lower than とか higher than のような言い方を使う場合もある。
  • |p| < |q| は、the absolute value of p is less than the absolute value of q と書くのが明確だけれど、ちょっと長ったらしい。短くしたければ、p is less than q in magnitude のように書いてもよい。
  • 小さい値から大きい値への変化 (つまり値の増加) を表す動詞としては、increase, rise, expand, scale up などが使える。が、increase 以外は、なんとなく、「正であることを前提とした、正の範囲での増加」を想像させてしまうかもしれないような気がする。
  • 大きい値から小さい値への変化 (つまり値の減少) を表す動詞としては、decrease, lessen, lower, fall, reduce, decline, drop, shrink, scale down などが使える。が、shrink と scale down は、「正であることを前提とした、正の範囲での減少」を想像させてしまうかもしれないような気がする。
  • より良い値への変化を表す動詞としては、improve が使える。この場合おそらく、「より良い値とは、より大きい値のことなのか、より小さい値のことなのか」は問わずに、いずれにせよ improve が使えるかと思う。
  • 「値が大きいほど良い」という場合に、より良い値 (より大きい値) への変化を表す動詞としては、enhance も使える。「値が小さいほど良い」という場合に、より良い値 (より小さい値) への変化を表したいときには、enhance は使いづらい (使うと誤解を招きかねない) ように思う。
  • より悪い値への変化を表す動詞としては、degrade, deteriorate などが使える。日本語だと「質が低下する」のように「低下」という言葉を使うけれど、「値が大きいほど質が悪い」という数値だってあるわけで、「低下」という字面だけを見て「値も小さくなるんだな」と決めつけてはならない (その決めつけに基づいて英語の動詞を選ぶと、多分混乱を招く) 。逆に、degrade, deteriorate などの単語を「悪化」ではなく「低下」と訳すと、暗黙に「値が小さいほど質が悪い」という印象を与えることになるかもしれないので、和訳の仕方も要注意 (文脈を見て、誤解の余地がないかどうかをよく確認する必要がある)。
  • 「圧縮率」も問題がある。文脈によって定義が違うので、文脈を見ないと危険である (人によって定義が違う、くらいに考えておくのが安全)。
    たとえば、「手法Aだと100MBのファイルが60MBに圧縮され、手法Bだと100MBのファイルが30MBに圧縮される」とする。この場合、圧縮の「性能」とか「効率」の「良し悪し」については、誰がどう見ても「手法Aより手法Bの方が良い」と言えるから、「手法Aを改善したものが手法Bである」のような言い方も可能だし、better とか improve とか superior といった単語も使える。ここまでは問題なし。
    でも、「手法Aの圧縮率」という言い方で「60%」を意図する人と「40%」を意図する人がいる。つまり、同じ「圧縮率」という言葉で、「圧縮後の大きさの、圧縮前の大きさに対する比率」を意味する人と、「圧縮によって削減された大きさの、圧縮前の大きさに対する比率」を意味する人がいる。あるいは、「5:3」という比、もしくは「5/3」という値を意味する人もいる (これは、「圧縮されたものを伸長したら何倍になるか」という値だと考えると分かりやすい)。このように「圧縮率」の定義には曖昧性がある (というか、定義の文脈依存性が極めて高い) ので、「手法Aと手法Bのどちらの方が圧縮率が大きい (高い) のか」や「圧縮率が大きい (高い) ほど性能が良いのか、それとも、圧縮率が小さい (低い) ほど性能が良いのか」については、文脈と定義を確認しないと、はっきりしたことが言えない。つまり迂闊に翻訳できない。くわばらくわばら。
